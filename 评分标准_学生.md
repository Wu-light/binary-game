## 二、进阶功能实现（30分）

### 2.1 查找最近存活祖先（10分）

**功能说明：**
当单位死亡时，需要找到最近的存活祖先节点用于子节点重连。

**评分要求：**

| 评分项 | 分值 | 详细说明 |
|-------|-----|---------|
| 遍历逻辑 | 4分 | 从parent开始向上遍历直到根节点 |
| 存活检查 | 3分 | 正确检查ancestor->isActive状态 |
| 终止条件 | 2分 | 找到存活祖先或到达根节点时停止 |
| 空指针处理 | 1分 | 遇到nullptr时正确返回 |

**正确实现示例：**
```cpp
StrategyNode* StrategyNode::findNearestAliveAncestor() {
    StrategyNode* ancestor = parent;
    
    // ✅ 向上遍历直到找到存活的祖先
    while (ancestor != nullptr) {
        if (ancestor->isActive && ancestor->unit != nullptr) {
            return ancestor;  // 找到存活祖先
        }
        ancestor = ancestor->parent;  // 继续向上
    }
    
    // ✅ 没有存活祖先，返回nullptr
    return nullptr;
}
```

**评分标准：**
- ✅ 完全正确：10分
- ⚠️ 逻辑错误：6-8分（遍历方向错误或条件错误）
- ⚠️ 缺少终止条件：4-6分（可能导致无限循环）
- ❌ 实现失败：0-3分（功能完全失效）

---

### 2.2 动态重连机制（10分）

**功能说明：**
单位死亡时自动将子节点重连到存活祖先节点，这是游戏核心机制！

**评分要求：**

| 评分项 | 分值 | 详细说明 |
|-------|-----|---------|
| 查找祖先调用 | 2分 | 正确调用findNearestAliveAncestor() |
| 左子节点挂载 | 3分 | 优先挂载到祖先的左子节点位置 |
| 右子节点挂载 | 3分 | 左子节点已满时挂载到右子节点 |
| 深度更新调用 | 2分 | 挂载后调用updateDepth更新子树深度 |

**关键逻辑：**
```cpp
void StrategyNode::handleUnitDeath() {
    // ✅ 查找存活祖先
    StrategyNode* ancestor = findNearestAliveAncestor();
    
    if (ancestor) {
        // ✅ 重连左子节点
        if (leftChild && leftChild->isActive) {
            if (!ancestor->leftChild) {
                ancestor->leftChild = leftChild;
                leftChild->parent = ancestor;
                updateDepth(leftChild, ancestor->depth + 1);
            } else if (!ancestor->rightChild) {
                ancestor->rightChild = leftChild;
                leftChild->parent = ancestor;
                updateDepth(leftChild, ancestor->depth + 1);
            }
        }
        
        // ✅ 重连右子节点（类似逻辑）
        // ...
    }
}
```

**评分标准：**
- ✅ 完全正确：10分（游戏正常运行）
- ⚠️ 部分实现：6-8分（重连逻辑不完整）
- ⚠️ 深度未更新：4-6分（重连成功但深度错误）
- ❌ 实现失败：0-3分（程序崩溃或树断裂）
