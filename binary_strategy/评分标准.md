# 双子战略 - 评分标准详解

## 一、评分系统概览

双子战略的评分系统设计既鼓励策略性决策，又奖励阵型管理和Root保护能力。评分系统与二叉树的核心概念紧密结合。

### 评分公式
```
总得分 = 波次得分 + Root保护奖励 + 阵型完整性奖励 + 快速击败奖励
```

## 二、详细评分项目

### 1. 波次基础得分 (Wave Base Score)
**计算公式: `波次数 × 100`**

每成功通过一个波次获得基础分数。

| 波次 | 基础得分 |
|-----|---------|
| 1   | 100分   |
| 2   | 200分   |
| 3   | 300分   |
| ... | ...     |

---

### 2. 存活单位奖励 (Surviving Unit Bonus)
**每存活单位: 50分**

奖励阵型管理能力，鼓励保护所有单位。

计算方式:
```
存活单位数量 × 50分
```

**示例计算:**
- 波次结束后存活5个单位: 5 × 50 = 250分
- 波次结束后存活8个单位: 8 × 50 = 400分

---

### 3. Root存活奖励 (Root Survival Bonus)
**固定分值: 200分（每波次）**

**获得条件: Root单位在波次结束时仍然存活**

这是最重要的评分项，体现了游戏核心机制：Root是阵型的核心。

**策略意义:**
- Root存活意味着整棵子树获得加成
- Root死亡导致子树所有单位失效
- 保护Root是最高优先级策略

---

### 4. 快速击败奖励 (Quick Victory Bonus)
**固定分值: 100分**

**获得条件: 该波次回合数少于10回合**

奖励高效的战斗策略和二叉树算法应用。

**获得难度分析:**
- 需要快速消灭敌人Root
- 合理利用单位攻击范围
- 优化的目标选择策略

---

### 5. Roguelike奖励选择质量评估

虽然不直接计入分数，但奖励选择影响后续波次的得分能力:

#### 单位奖励质量评估
- **骑士/坦克**: 高价值（可当Root）
- **弓箭手/法师**: 中等价值（远程攻击）
- **治疗者**: 辅助价值（持续作战）
- **刺客**: 特殊价值（对Root额外伤害）

#### Buff奖励价值
- **永久攻击加成**: 长期收益最高
- **永久防御加成**: 提升生存能力
- **生命恢复**: 持续作战保障
- **护盾**: 紧急保护手段

---

## 三、难度递增与得分关系

### 难度递增规则
```
敌人数量 = 波次数 + 2
敌人强度随波次递增
```

| 波次 | 敌人数量 | 预计难度 | 潜在得分 |
|-----|---------|---------|---------|
| 1   | 3个     | 简单    | ~350分  |
| 2   | 4个     | 中等    | ~500分  |
| 3   | 5个     | 困难    | ~650分  |
| 4   | 6个     | 挑战    | ~800分  |
| 5+  | 7+个    | 高难度  | 持续增长|

### 得分增长曲线
- 前期波次：快速得分，建立基础
- 中期波次：挑战增加，需要策略优化
- 后期波次：高风险高回报，考验Root保护能力

## 五、Root机制与评分的关系

### Root加成计算

Root存活时，整棵子树获得加成，加成随距离衰减:

```
加成百分比 = max(5, 30 - distanceFromRoot × 5)
```

| 距离Root | 加成百分比 | 示例效果 |
|---------|-----------|---------|
| 0（Root本身） | 30% | 攻击力+30% |
| 1（直接子兵） | 25% | 攻击力+25% |
| 2（孙兵）     | 20% | 攻击力+20% |
| 3            | 15% | 攻击力+15% |
| 4            | 10% | 攻击力+10% |
| 5+           | 5%  | 攻击力+5% |

### Root死亡的影响

Root死亡时：
1. **立即失去加成**: 所有子树单位失去Root加成
2. **战斗力下降**: 攻击力、防御力大幅降低
3. **可能连锁死亡**: 子树单位更容易被消灭
4. **得分大幅减少**: Root存活奖励清零

## 六、动态重连与评分影响

### 重连机制的价值

当单位死亡时，动态重连：
- **保持阵型完整**: 子节点自动挂载到存活祖先
- **减少损失**: 避免整棵子树失效
- **维持得分**: 存活单位继续获得奖励

### 重连策略优化

优秀的重连策略：
1. **选择最近祖先**: 减少距离衰减
2. **平衡左右分布**: 保持树结构平衡
3. **优先Root分支**: 尽量挂载到Root链上

## 七、阵型扩展与评分策略

### 扩展时机选择

| 阵型规模 | 优势 | 风险 | 建议 |
|---------|------|------|------|
| 4-6个单位 | 管理简单 | 战斗力有限 | 前期建议 |
| 7-10个单位 | 战斗力强 | Root压力大 | 中期优化 |
| 11+个单位 | 极强火力 | 极高风险 | 后期挑战 |

### 扩展策略建议

1. **前期快速扩展**
   - 选择骑士/坦克当Root
   - 快速添加子兵
   - 建立基础阵型

2. **中期稳定扩展**
   - 添加远程单位（弓箭手/法师）
   - 添加辅助单位（治疗者）
   - 平衡攻击和防御

3. **后期谨慎扩展**
   - 确保Root安全
   - 选择高价值单位
   - 避免过度扩张

## 九、游戏模式评分差异

### 主帅模式（高挑战）

**评分特点:**
- 主帅阵亡立即失败
- 得分积累中断
- 更高风险更高回报

### 全军模式（稳定）

**评分特点:**
- 全军覆没才失败
- 得分积累稳定
- 更低风险稳定回报

**最高得分潜力**: 较稳定（持续积累）

## 十二、评分系统代码实现

详见 `game_system.hpp` 中的 `calculateWaveScore()` 方法:

```cpp
int calculateWaveScore() {
    int waveScore = 0;
    
    // 基础波次分
    waveScore += wave * 100;
    
    // 存活单位奖励
    for (auto* node : playerUnits) {
        if (node->getUnit() && node->getUnit()->isUnitAlive()) {
            waveScore += 50;
        }
    }
    
    // Root存活奖励
    if (playerRoot && playerRoot->getUnit() && playerRoot->getUnit()->isUnitAlive()) {
        waveScore += 200;
    }
    
    // 快速击败奖励（回合数少）
    if (turn < 10) {
        waveScore += 100;
    }
    
    return waveScore;
}
```
