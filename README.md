# 双子战略 (Binary Strategy) - 游戏实现

## 项目概述

这是一个基于二叉树结构的策略战棋游戏，将二叉树数据结构应用于实时战术网络，而非仅用于数据存储。

## 核心概念

### 1. 二叉树阵型
- 每个单位（节点）最多有两个子节点
- 形成"根→子兵→孙兵"的树状阵型
- 动态扩展和重连机制

### 2. Root机制
- Root单位是树形结构的核心
- Root存活时，整棵子树获得增益效果
- 加成随树深度衰减
- Root被消灭时，子树所有单位立即失效

### 3. 动态重连逻辑
- 当某节点死亡时，自动将其子节点挂载到最近存活的祖先节点下
- 避免树断裂，保持阵型完整性

### 4. Roguelike机制
- 每波胜利后三选一奖励
- 随机兵种、事件、难度递增
- 无重复战局体验

## 游戏玩法

### 两种模式

1. **主帅模式**
   - 主帅阵亡直接失败
   - 消灭所有敌军获胜
   - 更具挑战性

2. **全军模式**
   - 4人开局
   - 可随时更换主帅
   - 消灭所有敌军获胜

### 游戏流程

1. **阵型设置阶段**
   - 初始化阵型结构
   - 配置Root单位

2. **战斗阶段**
   - 单位自动攻击
   - 处理死亡和重连
   - 检查胜负条件

3. **Roguelike奖励阶段**
   - 三选一奖励
   - 添加新单位或Buff
   - 进入下一波次

### 单位类型

- **K 骑士**: 高血量，近战，可当Root
- **A 弓箭手**: 远程，中等血量，精准射击
- **T 坦克**: 高防御，低攻击，可当Root
- **H 治疗者**: 辅助，治疗队友
- **S 刺客**: 高攻击，低血量，对Root额外伤害
- **M 法师**: 远程魔法，多目标攻击

### 操作方式

- **N** - 下一回合（手动战斗）
- **A** - 自动战斗模式
- **P** - 暂停游戏
- **1-3** - 选择Roguelike奖励

## 游戏策略

1. **保护Root单位** - Root存活时，整棵子树获得加成
2. **合理扩展阵型** - 平衡数量和质量
3. **优先击杀敌人Root** - Root死亡会导致敌方子树失效
4. **善用刺客** - 刺客对Root有双倍伤害
5. **治疗者很重要** - 保持队伍存活

## 评分系统

- **波次得分**: 波次数 × 100
- **存活单位**: 每个存活单位 +50分
- **Root存活**: +200分
- **快速击败**: 回合数<10 +100分

## 策略要点

### 阵型扩展策略
- 快速扩展树规模以增加战斗力
- 平衡"扩展"与"保护Root安全"
- 使用高血量单位担任Root

### Root保护策略
- Root是阵容核心，优先保护
- Root加成随距离衰减，合理布局
- Root死亡导致子树失效

### Roguelike奖励选择
- 单位奖励：增加阵型规模
- Buff奖励：提升现有单位能力
- 根据当前战局选择最优奖励

## 二叉树知识点映射

### 数据结构应用
- **链式二叉树**: 存储单位关系
- **指针操作**: 实时增删节点
- **动态重连**: 维护树结构完整性

### 算法应用
- **树遍历**: 查找目标、计算子树大小
- **深度计算**: Root加成衰减计算
- **路径查找**: 寻找最近存活祖先

### 算法复杂度
- 添加节点: O(1)
- 删除节点: O(depth)
- 重连操作: O(depth)
- 全树遍历: O(n)
